3.1 DARK CHANNEL PRIOR (DCP) FOR FOG DETECTION
---------------------------------------------------
Algorithm: Dark Channel Prior (DCP)
Paper: He et al., "Single Image Haze Removal via Dark Channel Prior" (2011)

CONCEPT:
- Images of outdoor scenes tend to have at least one color channel with very 
  low intensity in some region (the "dark channel")
- In hazy images, this property is weakened due to atmospheric scattering
- Fog density is measured by analyzing how much the dark channel property degrades

IMPLEMENTATION (fog_density.py):
1. Convert RGB image to floating point [0, 1]
2. For each pixel, compute minimum across all 3 channels
3. Apply morphological erosion with 15x15 kernel to get dark channel map
4. Find atmospheric light estimate from the brightest pixels in dark channel
5. Compute transmission map (shows how much light penetrates without scattering)
6. Fog density score = mean(dark_channel) * 100

OUTPUT:
- "dark_channel": Visual representation of dark regions (uint8 image)
- "transmission_map": How much light is attenuated (uint8 image)
- "fog_density": Numeric score 0-100 (0=clear, 100=densest fog)

KEY PARAMETERS:
- Window size: 15x15 pixels (for morphological operations)
- Omega (scattering coefficient): 0.95

3.2 YOLOV8 NANO FOR OBJECT DETECTION
-------------------------------------------
Model: YOLOv8n (Nano variant, ~3.2M parameters)
Framework: Ultralytics

CHARACTERISTICS:
- Lightweight version suitable for real-time inference
- Trained on COCO dataset (~80 classes)
- Input: 640x480 BGR frame
- Output: Bounding boxes with class confidence

CLASSES DETECTED:
- car, truck, bus, motorcycle, bicycle, person, dog, cat, etc.
- Traffic lights (with color classification: RED, GREEN, YELLOW)

INFERENCE PARAMETERS:
- Device: GPU (if available) or CPU
- Half precision: Enabled on CUDA devices
- Confidence threshold: Default from ultralytics

INTEGRATION:
- Model loaded once at startup (singleton pattern)
- Runs on every delivered frame (1 fps)
- Distance estimation: depth_heuristic = 50 / bbox_height (meters)

3.3 BRAKE LIGHT / RED GLOW DETECTION
----------------------------------------
Algorithm: HSV Color Space Thresholding

CONCEPT:
- Convert BGR frame to HSV (Hue, Saturation, Value)
- Define ranges for red color (accounting for hue wraparound at 0/180)
- Count red pixels in detected vehicle bounding boxes

HSV RED RANGES:
- Lower Red 1: H=0-10, S=120-255, V=70-255
- Upper Red 1: H=170-180, S=120-255, V=70-255
- (Two ranges because red wraps around in HSV hue space)

THRESHOLD:
- If >20% of bbox pixels are red, classify as brake light active

OUTPUT:
- "red_glow": Boolean (True if brake lights detected)
- "glow_boxes": List of bboxes with active brake lights
- "glow_frame": Annotated frame with red glow overlays

3.4 REVERSE GEOCODING (NOMINATIM)
--------------------------------------
Service: OpenStreetMap Nominatim API
Endpoint: https://nominatim.openstreetmap.org/reverse

PURPOSE:
- Convert GPS coordinates (lat, lon) to human-readable addresses
- Extract street name, road type, nearby landmarks

RATE LIMITING:
- 1 request per second (Nominatim terms of service)
- Cached for 10 seconds to avoid repeated calls
- Timeout: 10 seconds per request

FALLBACK:
- If geocoding fails, return "Unknown" for road name
- Continue pipeline without blocking

3.5 RISK SCORE CALCULATION
-------------------------------
Algorithm: Weighted Multi-Factor Risk Assessment

FORMULA:
  risk_score = Σ(weight_i × component_i)

where i ∈ {fog, distance, speed, road, red_glow, humidity}

WEIGHTS & FORMULAS:
1. Fog Component (0.30 weight)
   - normalized_fog = fog_density / 100  [0-1]

2. Distance Component (0.25 weight)
   - if distance = 0: score = 0
   - if distance ≤ 5m: score = 1 (collision zone)
   - linear interpolation from 5m to 60m
   - if distance ≥ 60m: score = 0

3. Speed Component (0.20 weight)
   - Inverted: lower recommended_speed = higher risk
   - Interpolate from 20 km/h (high risk) to 80 km/h (low risk)

4. Road Context Component (0.10 weight)
   - +0.4 if blackspots nearby
   - +0.3 if curve road detected
   - Maximum 1.0

5. Red Glow Component (0.10 weight)
   - 1.0 if red_glow is True
   - 0.0 if False

6. Humidity Component (0.05 weight)
   - placeholder (0 until sensor integrated)

THRESHOLDS:
- risk_score ≤ 0.30: "LOW"
- 0.30 < risk_score ≤ 0.60: "MEDIUM"
- risk_score > 0.60: "HIGH"

HARD OVERRIDE RULES:
- If red_glow AND distance < 20m → score ≥ 0.70 (HIGH)
- If distance < 10m → score ≥ 0.70 (HIGH)
- If curve AND distance < 25m → score ≥ 0.35 (MEDIUM floor)

3.6 OLLAMA LLM MODEL
------------------------
Model: Ollama Qwen3:1.7b (primary) or DeepSeek-r1:1.5b
Type: Quantized language model running locally (no API calls)
Size: ~1.7B parameters (fits on consumer hardware)
Inference: CPU/GPU based on available resources

PROMPTING:
- System role: "Advanced Driver Assistance System"
- Task: Analyze driving context and generate safety recommendations
- Output format: Structured text with numbered sections

LLM OUTPUT SECTIONS:
1. Risk Level - Summary assessment
2. Hazard Alert - Concise warning (extracted for voice)
3. Recommended Speed - Safe speed in km/h
4. Driving Suggestion - Action for driver
5. Short Explanation - Reason for recommendation
6. Object Detection Alert - Alert based on detected objects

LATENCY:
- First inference: ~2-3 seconds (model load + inference)
- Subsequent: ~1-2 seconds
- Cached for 5 seconds to avoid repeated calls

3.7 VOICE SYNTHESIS (PYTTSX3)
---------------------------------
Library: pyttsx3 (cross-platform TTS)
Engine: System default (SAPI5 on Windows, NSS on Linux, AVFoundation on macOS)

PARAMETERS:
- Rate: 160 wpm (words per minute)
- Volume: Default (system settings)
- Voice: Default system voice

THREADING:
- Runs in background thread (non-blocking)
- Does not interrupt UI or pipeline
- Multiple alerts can queue

COOLDOWN:
- 5 seconds per alert type (prevents alert spam)
- Tracked by alert_key: {"llm_recommendation", "fog", "distance", etc.}

EXTRACTION FROM LLM:
- Parse LLM output for "Hazard Alert:" line
- Extract only the alert text (first ~100 chars)
- If no structured output, fall back to full LLM text

4.1 fog_density.py
-------------------
FILE LOCATION: c:\Users\Puneeth Kumar\OneDrive\Desktop\IDP\Project\fodVision\fog_density.py

PURPOSE: Analyze fog density using Dark Channel Prior algorithm

DEPENDENCIES:
- cv2 (OpenCV)
- numpy
- Union from typing

KEY FUNCTIONS:

get_dark_channel(img, window_size=15)
  Input: img (float64 array, normalized [0,1])
         window_size (int, default 15)
  Output: dark_channel (2D array)
  Algorithm:
    - Compute minimum across 3 color channels for each pixel
    - Apply morphological erosion with window_size×window_size kernel
    - Return eroded minimum channel

get_atmospheric_light(img, dark_channel)
  Input: img (float64 array)
         dark_channel (2D array from above)
  Output: atmospheric_light (1D array, shape (3,) for BGR)
  Algorithm:
    - Find brightest 0.1% of pixels in dark channel
    - Average their RGB values to estimate atmospheric scattering

get_transmission_map(img, atmospheric_light, window_size=15, omega=0.95)
  Input: img (float64 array)
         atmospheric_light (1D array)
         window_size (int)
         omega (float, scattering coefficient)
  Output: transmission_map (2D array, [0-1])
  Algorithm:
    - Normalize image by dividing by atmospheric light
    - Compute dark channel of normalized image
    - transmission = 1 - omega * dark_channel
    - (Higher omega = more aggressive dehazing)

_prepare_image(image_or_path: Union[str, np.ndarray]) -> np.ndarray
  Input: Either file path string or numpy array
  Output: BGR image as uint8 numpy array
  Purpose: Handle both file and array inputs

estimate_fog_density(image_or_path: Union[str, np.ndarray])
  Input: Either image file path or BGR numpy array
  Output: Dictionary with keys:
    - "dark_channel": uint8 visualization (0-255)
    - "transmission_map": uint8 visualization (0-255)
    - "fog_density": float (0-100)
  Main function: Orchestrates DCP algorithm

USAGE EXAMPLE:
  from fog_density import estimate_fog_density
  import cv2
  
  # Method 1: From file
  result = estimate_fog_density("path/to/image.jpg")
  
  # Method 2: From numpy array
  frame = cv2.imread("image.jpg")
  result = estimate_fog_density(frame)
  
  fog_pct = result["fog_density"]  # e.g., 65.4

OUTPUT FORMAT:
  {
    "dark_channel": numpy.ndarray (uint8, shape=(H,W)),
    "transmission_map": numpy.ndarray (uint8, shape=(H,W)),
    "fog_density": float (0-100)
  }

4.2 dehaze.py
--------------
FILE LOCATION: c:\Users\Puneeth Kumar\OneDrive\Desktop\IDP\Project\fodVision\dehaze.py

PURPOSE: Enhance foggy images by removing atmospheric haze

KEY CLASS:

class DehazeModel
  __init__()
    - Initializes Dark Channel Prior dehazing model
    
  process(frame)
    Input: BGR frame (numpy array)
    Output: Dehazed BGR frame (numpy array, same shape)
    Algorithm:
      1. Convert to float [0,1]
      2. Compute dark channel
      3. Estimate atmospheric light
      4. Compute transmission map
      5. Apply dehazing formula: I(x) = max(J(x), t0)
         where J(x) = (I(x) - A) / transmission + A
      6. Clip to [0,1], convert back to uint8
    
USAGE:
  from dehaze import DehazeModel
  
  dehaze = DehazeModel()  # Create once (expensive)
  enhanced_frame = dehaze.process(foggy_frame)

4.3 object_detect.py
---------------------
FILE LOCATION: c:\Users\Puneeth Kumar\OneDrive\Desktop\IDP\Project\fodVision\object_detect.py

PURPOSE: Detect objects and estimate distances using YOLOv8

GLOBAL STATE:
- model: YOLO("models/yolov8n.pt") - loaded once
- _HAS_CUDA: Boolean, True if CUDA available
- _DEVICE: 0 (GPU) or "cpu"

KEY FUNCTIONS:

detect_traffic_light_color(roi)
  Input: roi (BGR image crop around traffic light)
  Output: "RED" | "GREEN" | "YELLOW" | None
  Algorithm:
    - Convert to HSV
    - Define color ranges for each light
    - Create masks for each color
    - Return color with highest pixel count
  
  Color Ranges (HSV):
    RED: H 0-10 or 170-180, S 120-255, V 70-255
    GREEN: H 40-90, S 40-255, V 40-255
    YELLOW: H 15-35, S 150-255, V 150-255

process_frame(frame)
  Input: frame (BGR image, processed by dehaze)
  Output: Tuple of:
    - annotated_frame (frame with bounding boxes drawn)
    - detections_list (list of detection dicts)
  
  Algorithm:
    1. Run YOLO inference on frame
    2. For each detection:
       a. Extract bounding box coordinates
       b. Get class label and confidence
       c. If traffic light: analyze ROI for color
       d. Draw box with appropriate color
       e. Calculate distance: 50.0 / max(height, 1)
       f. Add to detections list
    3. Return annotated frame and detections

DETECTION OUTPUT FORMAT:
  {
    "label": str (class name),
    "confidence": float (0-1),
    "bbox": [x1, y1, x2, y2] (pixel coordinates),
    "traffic_light_color": str ("RED"/"GREEN"/"YELLOW") or None,
    "distance": float (meters, heuristic estimate)
  }

DISTANCE HEURISTIC:
- Formula: distance = 50 / bbox_height
- Assumes fixed focal length and known object size
- Rough estimate (±30% error typical)
- Value 0 when height = 0 (prevents div by zero)

4.5 risk_score.py
------------------
FILE LOCATION: c:\Users\Puneeth Kumar\OneDrive\Desktop\IDP\Project\fodVision\risk_score.py

PURPOSE: Calculate multi-factor risk score and classification

KEY FUNCTIONS:

_normalise(value, lo, hi)
  Input: value (numeric), lo (lower bound), hi (upper bound)
  Output: float in [0, 1]
  Formula: (value - lo) / (hi - lo), clamped to [0, 1]

_normalise_inv(value, lo, hi)
  Input: value, lo, hi
  Output: float in [0, 1]
  Formula: 1 - _normalise(value, lo, hi)
  Purpose: Invert scale (lower value = higher risk)

compute_risk(fog_density, distance_to_nearest, recommended_speed, 
             road_context, red_glow, humidity=0.0)
  Input:
    - fog_density: 0-100 (%)
    - distance_to_nearest: meters (0 if no object)
    - recommended_speed: km/h (from fog module)
    - road_context: dict with "blackspots", "road_type"
    - red_glow: boolean
    - humidity: 0-100 (%) - currently unused
  
  Output: Dictionary with:
    - "risk_score": float 0-1
    - "risk_level": str ("LOW" | "MEDIUM" | "HIGH")
    - "component_scores": dict of individual scores
    - "hard_override": bool (True if override rule triggered)
    - "override_reason": str (explanation)
  
  Algorithm:
    1. Compute component scores:
       - fog: normalise(fog_density, 0, 100)
       - distance: _normalise_inv(distance, 5, 60)
       - speed: _normalise_inv(recommended_speed, 20, 80)
       - road: blackspot_bonus (0.4) + curve_bonus (0.3)
       - red_glow: 1.0 if True, 0.0 if False
       - humidity: normalise(humidity, 0, 100)
    
    2. Apply weights:
       raw_score = sum(weight[i] * score[i] for all i)
       weights = {fog: 0.30, distance: 0.25, speed: 0.20,
                  road: 0.10, red_glow: 0.10, humidity: 0.05}
    
    3. Apply hard override rules:
       - If red_glow AND 0 < distance < 20: score ≥ 0.70
       - If 0 < distance < 10: score ≥ 0.70
       - If "curve" in road_type AND 0 < distance < 25: score ≥ 0.35
    
    4. Classify:
       - score ≤ 0.30: "LOW"
       - 0.30 < score ≤ 0.60: "MEDIUM"
       - score > 0.60: "HIGH"
    
    5. Return all scores and classification

get_road_context(lat: float, lon: float) -> Dict[str, Any]
  Input: 
    - lat: GPS latitude
    - lon: GPS longitude
  
  Output: Dictionary with:
    - "latitude": float
    - "longitude": float
    - "road": str (street name)
    - "road_type": str ("Highway" | "Urban Road")
    - "blackspots": list of nearby danger zones
    - "hazard_types": sorted list of hazard classifications
    - "raw_location": dict (full Nominatim response)

    1. REVERSE GEOCODING:
       - Query Nominatim API: 
         https://nominatim.openstreetmap.org/reverse?format=jsonv2&lat={lat}&lon={lon}
       - Extract street name from response
       - Handle errors gracefully (try/except with timeout)
    
    2. ROAD TYPE CLASSIFICATION:
       - If "highway" or "nh" in road name → "Highway"
       - Else → "Urban Road"
    
    3. BLACKSPOT DETECTION:
       - For each blackspot in config:
         - Calculate distance using geodesic (Haversine)
         - If distance < 2.0 km:
           - Add to nearby_blackspots list
           - Extract hazard types:
             * fog_prone → add "fog_prone"
             * rain_prone → add "rain_prone"
             * landslide_prone → add "landslide_prone"
             * accident_prone → add "accident_prone"
             * weather_risk → add weather type
       - If road type is "curve": add "curve"
       - If expressway/highway: add "high_speed"
       - If no hazards: add "standard"
    
    4. RETURN context dict

BLACKSPOT FIELDS (from JSON):
  {
    "name": str,
    "state": str,
    "latitude": float,
    "longitude": float,
    "road_type": str,

PURPOSE: Generate and speak safety alerts using TTS

GLOBAL STATE:
- last_alert_time: dict tracking last speak time per alert_key
- ALERT_COOLDOWN: 5 seconds

KEY FUNCTIONS:

_speak(message)
  Input: message (str to speak)
  Purpose: Internal function to do TTS
  Algorithm:
    1. Initialize pyttsx3 engine
    2. Set rate to 160 wpm
    3. Say message text
    4. Wait until done (blocking)
  Thread: Called in background thread

speak_alert(alert_key, message)
  Input: 
    - alert_key: str (identifier for alert type)
    - message: str (text to speak)
  
  Algorithm:
    1. Get current time
    2. Check if alert_key in last_alert_time:
       - If yes and elapsed < ALERT_COOLDOWN: RETURN (skip)
       - Else: update last_alert_time[alert_key]
    3. Print "ALERT: {message}" to console
    4. Spawn background thread running _speak(message)
       - daemon=True (doesn't block program exit)
       - target=_speak, args=(message,)
  
  Purpose: Prevent alert spam with cooldown mechanism

USAGE:
  from voice_alert import speak_alert
  
  speak_alert("fog_warning", "Dense fog detected. Reduce speed.")
  # Same key won't trigger again for 5 seconds

ALERT KEYS USED IN PIPELINE:
  - "llm_recommendation": LLM-extracted hazard alert
  - "fog": Fog density warning
  - "distance": Distance to object warning
  - "red_glow": Brake light detected
  - "blackspot": Accident-prone area nearby
  - Others defined in old rule_engine (now disabled)

4.9 video_engine.py
---------------------
FILE LOCATION: c:\Users\Puneeth Kumar\OneDrive\Desktop\IDP\Project\fodVision\video_engine.py

PURPOSE: Extract frames at 1 FPS from video files (or webcam)

KEY CLASS:

class VideoFrame (dataclass)
  Attributes:
    - frame: cv2.Mat (BGR image)
    - timestamp: float (Unix time)
    - frame_index: int (absolute frame counter in file)
    - second_index: int (which 1-second bucket this frame belongs to)

class VideoEngine
  __init__(source=None, prefer_file=str, target_size=(960,540))
    Input:
      - source: video file path or camera index (None auto-selects)
      - prefer_file: preferred file path if source is None
      - target_size: tuple (width, height) for resize
    
    Initialization:
      1. Determine source (file or camera)
      2. Open with cv2.VideoCapture(source)
      3. Detect FPS from source
      4. Calculate frames to skip for 1 fps output
      5. Initialize frame counters
  
  Methods:
    is_opened: property (bool) - returns whether capture is open
    source_fps: property (float) - FPS of source
    skip_frames: property (int) - how many frames to skip
    
    read() -> VideoFrame | None
      Algorithm:
        1. Read next frame from source
        2. Count frames and determine if we should deliver this one
        3. For 1 fps delivery:
           - Skip (fps - 1) frames
           - Deliver every fps-th frame
        4. Resize to target_size
        5. Cache as last_delivered (for Streamlit reruns)
        6. Return VideoFrame with all metadata
        7. If source exhausted: return None
      
      Returns:
        - VideoFrame: next 1-fps frame
        - None: no new second-boundary frame yet (between seconds)

FRAME SAMPLING:
- 30 fps source video:
  Read frames: 0, 1, 2, ..., 29 (skip 0-28)
  Deliver: frame 29 (end of second 0)
  Then: frame 59, 89, 119, ... (end of each second)

TARGET SIZE:
- Default: (960, 540) - then resized to (640, 480) in pipeline

  - Input size: 640×480
  - Confidence threshold: default ultralytics
  - Device: GPU if available, else CPU
  - Half precision: enabled on CUDA

DISTANCE ESTIMATION:
  - Formula: distance = 50 / max(bbox_height, 1)
  - Units: meters

BRAKE LIGHT DETECTION:

  - HSV red ranges: [0-10, 170-180] hue

RISK SCORE WEIGHTS:

HARD OVERRIDE RULES:

8. INTEGRATION POINTS

STREAMLIT ↔ PIPELINE:
  - DashBoard.py imports ADASPipeline class
  - Creates singleton in session_state
  - Calls process(frame, lat, lon) every second
  - Displays result dict in UI components

VIDEO ENGINE ↔ STREAMLIT:
  - VideoEngine created in session_state
  - DashBoard calls engine.read() in loop
  - Returns None between seconds (Streamlit reruns)
  - Caches last_delivered for display consistency

PIPELINE ↔ MODULES:
  - pipeline.py imports all inference modules
  - Calls each module in sequence (orchestrator pattern)
  - Passes outputs as inputs to next stage
  - Exception handling: print and continue with defaults

FOG & DEHAZE:
  - fog_density.py operates on raw frame
  - dehaze.py operates on same raw frame
  - Both use Dark Channel Prior (shared algorithm)
  - Could optimize by sharing computation (not yet implemented)

DETECTION ↔ RED GLOW:
  - object_detect.py returns detections + annotated_frame
  - red_glow.py takes both as input
  - Adds glow overlays to already-annotated frame
  - Final frame has both object boxes + glow indicators

RISK ↔ CONTEXT:
  - Road context gated at 10s, cached
  - Risk score uses cached context
  - Updated context sent to LLM (5s gating)

LLM ↔ VOICE:
  - pipeline.py calls get_llm_decision()
  - Extracts alert text via _extract_alert_from_llm()
  - Passes to speak_alert() with unique alert_key
  - Cooldown prevents repeated voice alerts

NOMINATIM API:
  - Called from road_context.py
  - Timeout: 10 seconds
  - Rate limited: 1 request per second (Nominatim ToS)
  - Fallback: return "Unknown" if failure

OLLAMA SERVICE:
  - Must be running locally (ollama serve)
  - Called via ollama.chat() library
  - Model preloaded in Ollama daemon
  - Returns structured text response
